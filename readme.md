# 最优化实验室 (OptLab)

> 基于 Web 的交互式最优化理论教学实验平台

[![Python Version](https://img.shields.io/badge/python-3.13+-blue)](https://www.python.org/)
[![FastAPI](https://img.shields.io/badge/FastAPI-0.128+-green)](https://fastapi.tiangolo.com/)
[![D3.js](https://img.shields.io/badge/D3.js-v7-orange)](https://d3js.org/)

## 项目简介

OptLab 是一个面向《最优化理论与方法》课程的在线教学实验平台，旨在通过交互式可视化降低最优化算法的学习门槛。平台提供「教学中心」和「实验中心」两大功能模块，支持算法流程动态演示、参数交互调节、实验记录持久化及 AI 助教辅助功能。

本项目适用于计算机设计比赛「微课与教学辅助」赛道。

## 功能特性

### 1. 教学中心

提供一系列算法实验，以已实现的一维搜索方法为例：

- **区间收缩法**：黄金分割法、斐波那契数列法、二分法的可视化对比 (已完成)
- **点搜索法**：梯度下降法、牛顿法、割线法等单点迭代算法 (已完成)
- **应用实践**：一维搜索在实际问题中的应用 (已完成)
- **线性规划**：单纯形法流程演示 (已上线)
- **SVM 优化**：核技巧三维可视化（CSV上传、升维动画、超平面自动拟合、实时准确率）(已上线)

每个实验包含：
- 基于 D3.js 绘制的动态函数图像
- 迭代点/迭代区间的可视化标注
- 参数滑条（初始区间、精度、迭代次数等）
- 播放/暂停/单步/重置控制面板
- 自定义函数输入（Math.js 表达式解析）

SVM 核技巧实验包含：
- Three.js 三维点云与可移动超平面
- 多项式核 / 高斯核参数切换（参数区随核函数自动切换）
- 预设数据集 + CSV 上传 + 列映射（x/y/label）
- 自动超平面拟合与实时划分准确率

### 2. 实验中心

实验中心的目的是将教学中心已实现的各类控件抽象出来，允许用户组合使用：

- 输入目标函数、决策变量、约束条件
- 选择可视化面板类型
- 零代码搭建自定义实验
- 上传实验指导书、分享实验

### 3. 用户系统与实验记录

- 用户注册/登录（JWT 认证）
- 实验数据自动保存
- 一键导出实验报告（PDF/Markdown）

### 4. AI 助教（核心亮点）

基于 LangChain Agent 实现的智能助教，支持以下功能：

| 功能 | 描述 |
|------|------|
| 查看实验指导书 | 向后端请求预置的实验指导书文档 |
| 高亮网页组件 | 通过 JavaScript 改变控件外观，引导用户交互 |
| 查看实验数据 | 查询当前网页上的实时实验数据（迭代次数、当前区间等） |

用户可通过侧边对话栏与 AI 助教交互，AI 助教会在对话框中说明实验目标和步骤，并高亮需要互动的控件。

### 三、技术风险评估

| 风险点 | 等级 | 缓解措施 |
|--------|------|----------|
| D3.js 大规模数据渲染性能 | 低 | 当前 1000 采样点足够；必要时可切换 Canvas 渲染 |

## 快速开始

### 环境要求

- Python 3.13+
- PostgreSQL 14+（可选，开发环境可用 SQLite）

### 安装

```bash
# 1. 克隆项目
git clone <repository-url>
cd OptLab

# 2. 创建虚拟环境
python -m venv .venv
source .venv/bin/activate  # Linux/macOS
# .venv\Scripts\activate   # Windows

# 3. 安装依赖
pip install -e .

# 4. 配置环境变量（见下方说明）

# 5. 运行开发服务器
uvicorn main:app --reload --port 8001
```

### 配置 AI 助手

项目使用 DeepSeek 大模型为流程观察页面提供 AI 操作引导。

**第一步：获取 API Key**

前往 [DeepSeek 开放平台](https://platform.deepseek.com/api_keys) 注册并创建 API Key。

**第二步：编辑 `.env` 文件**

项目根目录下的 `.env` 文件包含所有可配置项：

```env
# DeepSeek API 配置
# 从 https://platform.deepseek.com/api_keys 获取（必填）
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx

# 模型名称（可选，默认 deepseek-chat）
DEEPSEEK_MODEL=deepseek-chat

# 模型温度（可选，默认 0.7，范围 0-2）
DEEPSEEK_TEMPERATURE=0.7
```

| 变量 | 必填 | 说明 |
|------|------|------|
| `DEEPSEEK_API_KEY` | ✅ | DeepSeek API 密钥 |
| `DEEPSEEK_MODEL` | ❌ | 模型名称，默认 `deepseek-chat` |
| `DEEPSEEK_TEMPERATURE` | ❌ | 生成温度，值越低回答越确定，默认 `0.7` |

> ⚠️ `.env` 已被 `.gitignore` 忽略，API Key 不会提交到仓库。

└─────────────────────────────────────────────────────────────────────┘

#### 各部分说明

| 部分 | 位置 | 作用 |
|------|------|------|
| **页面配置** (AI_CONFIG) | 前端 `window.AI_CONFIG` | 包含页面 ID、按钮注册表和指导书，由 `ai-sidebar.js` 自动读取并发送给后端 |
| **实验文档** (docs/) | 后端 `docs/实验指导书/` + `docs/公式/` | 实验步骤说明和算法公式，由后端根据 `page_id` 自动加载 |
| **侧栏组件** | `templates/components/_ai_sidebar.html` | 解耦后的侧栏 HTML 结构，通过 `base.html` 统一引入 |
| **交互逻辑** | `static/js/common/ai-sidebar.js` | 处理 HTTP 请求(`/api/assistant/chat`)、页面高亮定位及对话渲染 |
| **响应 Schema** | 后端 `schemas/agent.py` | Pydantic 模型 `AssistantSchema`，强制 AI 输出 `text` + `highlight_ids` |

#### page_id 与文档映射

在 `core/agent.py` 的 `_DOCS_MAP` 中定义映射关系：

| page_id | 实验指导书 | 算法公式 |
|---------|-----------|---------|
| `range_search_observation` | `区间收缩法-流程观察.txt` | 黄金分割法、斐波那契数列法、二分法 |
| `point_search_observation` | `点搜索-流程观察.txt` | 梯度下降法、牛顿法、割线法 |
| `svm_smo_index` | `SVM-SMO-课程主页.txt` | SVM基础概念 |
| `svm_smo_kernel_trick` | `SVM-核技巧-三维可视化.txt` | SVM核技巧 |

新增实验页面时，只需在 `_DOCS_MAP` 中添加对应的条目，并在 `docs/` 下放置文档文件即可。

#### 为新页面添加 AI 助手

在 AI 侧栏功能解耦后，为一个新网页添加 AI 助手只需以下步骤：

**1. 继承基础母版** — 使用 `{% extends 'base.html' %}`，确保自动引入侧栏组件。

**2. 准备文档** — 在 `docs/` 下添加实验指导书和算法公式文件，并在 `core/agent.py` 的 `_DOCS_MAP` 中注册映射关系。

**3. 配置页面参数** — 在页面 `{% block scripts %}` 中定义 `window.AI_CONFIG`：

```javascript
{% block scripts %}
<script>
    window.AI_CONFIG = {
        PAGE_ID: 'your_page_id',
        BUTTON_REGISTRY: [
            { id: "btn-1", description: "按钮1描述", type: "normal" },
            // ...
        ],
        GUIDEBOOK: `页面详细描述和操作指南...`
    };
</script>
{% endblock %}
```

详细操作指南请参考 `docs/侧栏添加.md`。

> `type` 字段目前默认为 `"normal"`，预留用于未来区分控件类型（如 `"slider"`、`"modal"` 等）以实现差异化交互引导。
