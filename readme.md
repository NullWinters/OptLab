# 最优化实验室 (OptLab)

> 基于 Web 的交互式最优化理论教学实验平台

[![Python Version](https://img.shields.io/badge/python-3.13+-blue)](https://www.python.org/)
[![FastAPI](https://img.shields.io/badge/FastAPI-0.128+-green)](https://fastapi.tiangolo.com/)
[![D3.js](https://img.shields.io/badge/D3.js-v7-orange)](https://d3js.org/)

## 项目简介

OptLab 是一个面向《最优化理论与方法》课程的在线教学实验平台，旨在通过交互式可视化降低最优化算法的学习门槛。平台提供「教学中心」和「实验中心」两大功能模块，支持算法流程动态演示、参数交互调节、实验记录持久化及 AI 助教辅助功能。

本项目适用于计算机设计比赛「微课与教学辅助」赛道。

---

## 功能特性

### 1. 教学中心

提供一系列算法实验，以已实现的一维搜索方法为例：

- **区间收缩法**：黄金分割法、斐波那契数列法、二分法的可视化对比
- **点搜索法**：牛顿法、割线法等单点迭代算法
- **应用实践**：综合应用场景

每个实验包含：
- 基于 D3.js 绘制的动态函数图像
- 迭代点/迭代区间的可视化标注
- 参数滑条（初始区间、精度、迭代次数等）
- 播放/暂停/单步/重置控制面板
- 自定义函数输入（Math.js 表达式解析）

### 2. 实验中心

实验中心的目的是将教学中心已实现的各类控件抽象出来，允许用户组合使用：

- 输入目标函数、决策变量、约束条件
- 选择可视化面板类型
- 零代码搭建自定义实验
- 上传实验指导书、分享实验

### 3. 用户系统与实验记录

- 用户注册/登录（JWT 认证）
- 实验数据自动保存
- 一键导出实验报告（PDF/Markdown）

### 4. AI 助教（核心亮点）

基于 LangChain Agent 实现的智能助教，支持以下功能：

| 功能 | 描述 |
|------|------|
| 查看实验指导书 | 向后端请求预置的实验指导书文档 |
| 高亮网页组件 | 通过 JavaScript 改变控件外观，引导用户交互 |
| 查看实验数据 | 查询当前网页上的实时实验数据（迭代次数、当前区间等） |

用户可通过侧边对话栏与 AI 助教交互，AI 助教会在对话框中说明实验目标和步骤，并高亮需要互动的控件。

## 软件可行性分析

### 一、现有基础评估

项目当前已具备以下技术能力：

- ✅ FastAPI 基础路由架构
- ✅ D3.js 动态可视化引擎（函数图像绘制、迭代点标注、缩放平移）
- ✅ Math.js 表达式解析（支持自定义函数输入）
- ✅ 参数交互控制（滑条、播放控制）
- ✅ 响应式布局（Tailwind CSS）

**结论：技术选型合理，后续功能开发具有坚实的可行性基础。**

### 二、各模块可行性分析

#### 1. 教学中心（算法实验可视化）

| 子功能 | 技术方案 | 可行性 |
|--------|----------|--------|
| 函数图像绘制 | D3.js SVG 绑定 + 比例尺 | ✅ 已实现 |
| 迭代点动画 | D3 transition + setInterval | ✅ 已实现 |
| 参数滑条控制 | HTML5 input[type=range] + 事件监听 | ✅ 已实现 |
| 自定义函数解析 | Math.js evaluate() | ✅ 已实现 |
| 多算法对比 | 复用同一 Visualization 类 | ✅ 已实现 |

#### 2. 数据持久化

| 子功能 | 技术方案 | 可行性 |
|--------|----------|--------|
| 用户认证 | FastAPI + JWT (python-jose) | ✅ 完全可行 |
| 数据库 | PostgreSQL + SQLAlchemy async | ✅ 完全可行 |
| 实验记录存储 | JSONB 字段存储迭代数据 | ✅ 完全可行 |

#### 3. AI 助教功能

这是本项目的核心亮点，技术架构如下：

```
┌─────────────┐    WebSocket    ┌─────────────┐    LangChain    ┌─────────────┐
│   前端页面   │ ──────────────► │   FastAPI   │ ──────────────► │   LLM API   │
│  (对话 UI)   │ ◄────────────── │  (Agent)    │ ◄────────────── │ (OpenAI/...) │
└─────────────┘                 └─────────────┘                 └─────────────┘
       │                               │
       │   ┌─────────────────────────┐  │
       │   │    Tool Calling         │  │
       │   │ - VIEW_GUIDE (查指导书)  │  │
       │   │ - HIGHLIGHT (高亮组件)   │  │
       │   │ - GET_DATA (查实验数据)  │  │
       │   └─────────────────────────┘  │
       ▼                                ▼
┌─────────────┐                 ┌─────────────┐
│ JavaScript │                 │  后端 API   │
│ DOM 操作   │                 │  文档读取   │
└─────────────┘                 └─────────────┘
```

| 功能 | 技术实现 | 可行性 |
|------|----------|--------|
| 查看实验指导书 | 预置 Markdown 文档 → 向量检索/直接读取 → 返回给 Agent | ✅ 完全可行 |
| 高亮网页组件 | Agent 返回 JSON 指令 → 前端 `document.querySelector()` + 修改 CSS 类 | ✅ 完全可行 |
| 查看实验数据 | 前端暴露 `window.getExperimentState()` → Agent 调用返回 JSON | ✅ 完全可行 |

**Prompt 设计示例：**

```
你是一个最优化实验的 AI 助教。用户当前在"区间收缩法"实验页面。

当用户请求以下操作时，按格式返回：
- 查看指导书 → 返回 "TOOL:view_guide:{指南ID}"
- 高亮控件 → 返回 "TOOL:highlight:{CSS选择器}"
- 查看数据 → 返回 "TOOL:get_data"
- 其他问题 → 直接用自然语言回答

记住：用简洁的语言引导用户完成实验。
```

#### 4. 实验中心（自定义实验）

| 子功能 | 技术方案 | 可行性 |
|--------|----------|--------|
| 实验配置存储 | JSON Schema → 数据库 | ✅ 完全可行 |
| 动态渲染 | 前端根据 JSON 配置生成表单 | ✅ 完全可行 |
| 公式解析 | 复用现有 Math.js | ✅ 完全可行 |

**推荐策略**：MVP 阶段采用「预设模板填充」模式，用户只需修改参数而非从零创建；后续版本再开放完全自定义。

### 三、技术风险评估

| 风险点 | 等级 | 缓解措施 |
|--------|------|----------|
| D3.js 大规模数据渲染性能 | 低 | 当前 1000 采样点足够；必要时可切换 Canvas 渲染 |
| LangChain Agent 响应稳定性 | 中 | 限制 Tool 数量；设计清晰的 Prompt 边界 |
| PostgreSQL 部署 | 低 | 提供 Docker Compose 一键部署 |
| 前端状态管理复杂度 | 中 | 采用 ES6 模块化 + 单例模式，避免引入重型框架 |

## 软件需求分析

### 一、用户角色

| 角色 | 描述 | 权限 |
|------|------|------|
| 学生 | 使用教学中心进行实验 | 浏览实验、修改参数、保存记录 |
| 教师/内容创建者 | 创建实验、上传指导书 | 全部权限 + 实验管理 |
| 管理员 | 系统管理 | 用户管理、系统配置 |

### 二、功能需求

#### 2.1 教学中心

- **F1**：展示算法实验列表，支持分类筛选
- **F2**：加载实验页面，渲染函数图像
- **F3**：参数滑条调节，实时更新可视化
- **F4**：播放/暂停/单步/重置控制
- **F5**：自定义函数输入与解析
- **F6**：保存实验参数与结果到个人记录

#### 2.2 实验中心

- **F7**：创建新实验，填写基本信息（名称、描述）
- **F8**：配置目标函数、变量、约束
- **F9**：选择可视化面板组件
- **F10**：保存实验配置到数据库
- **F11**：分享实验链接（公开/私密）
- **F12**：导入他人共享的实验

#### 2.3 用户系统

- **F13**：用户注册（邮箱/学号）
- **F14**：用户登录（JWT Token）
- **F15**：查看个人实验历史
- **F16**：导出实验报告（PDF）

#### 2.4 AI 助教

- **F17**：侧边对话栏交互
- **F18**：查看当前实验指导书
- **F19**：高亮引导用户操作
- **F20**：查询实时实验数据
- **F21**：自然语言问答（算法原理、参数含义等）

### 三、非功能需求

| 需求 | 描述 |
|------|------|
| **性能** | 页面加载 < 2s；可视化交互延迟 < 100ms |
| **可用性** | 界面简洁，适合教学场景；支持移动端浏览 |
| **可维护性** | 模块化前端代码；清晰的 API 文档 |
| **安全性** | 用户密码哈希存储；API 鉴权；输入过滤 |

## 快速开始

### 环境要求

- Python 3.13+
- PostgreSQL 14+（可选，开发环境可用 SQLite）

### 安装

```bash
# 1. 克隆项目
git clone <repository-url>
cd OptLab

# 2. 创建虚拟环境
python -m venv .venv
source .venv/bin/activate  # Linux/macOS
# .venv\Scripts\activate   # Windows

# 3. 安装依赖
pip install -e .

# 4. 运行开发服务器
uvicorn main:app --reload
```

### 访问

打开浏览器访问 http://localhost:8000
