在 AI 侧栏功能解耦后，为一个新网页添加 AI 侧栏助手非常简单，主要分为三个步骤：**继承母版**、**定义页面内容** 和 **配置 AI 参数**。

以下是具体操作流程及代码示例：

### 1. 继承基础母版 (base.html)
新网页必须使用 Jinja2 的模板继承机制。通过继承 `base.html`，网页会自动引入所需的 CSS 样式（`ai-sidebar.css`）、HTML 结构（`_ai_sidebar.html`）以及交互逻辑（`ai-sidebar.js`）。

```html
{% extends 'base.html' %}

{% block title %}页面标题{% endblock %}
```

### 2. 设置页面交互元素
在 `{% block content %}` 中编写网页的 HTML 内容。为了让 AI 能够识别并高亮页面上的控件（如按钮、滑块、下拉框等），必须为这些元素设置唯一的 `id`。

```html
{% block content %}
<div class="container">
    <select id="algorithm-select">...</select>
    <button id="start-btn">开始实验</button>
    <input type="range" id="param-slider">
</div>
{% endblock %}
```

### 3. 配置 window.AI_CONFIG
在 `{% block scripts %}` 中定义 `window.AI_CONFIG` 对象。这是 AI 侧栏工作的核心，它告诉后端 Agent 当前页面的上下文信息。

该对象包含三个关键字段：
*   **`PAGE_ID`**: 页面唯一标识符。
*   **`BUTTON_REGISTRY`**: 可高亮控件列表。每个对象包含 `id`（对应 HTML 元素的 ID）、`description`（描述该控件的功能）和 `type`（类型，通常填 `"normal"`）。
*   **`GUIDEBOOK`**: 网页的详细描述、布局说明及使用流程。这部分信息将发送给 LLM，作为其推理操作方法的依据。

```html
{% block scripts %}
<script>
    window.AI_CONFIG = {
        PAGE_ID: 'your_page_unique_id',
        BUTTON_REGISTRY: [
            { id: "algorithm-select", description: "算法选择下拉框", type: "normal" },
            { id: "start-btn", description: "开始实验按钮", type: "normal" },
            { id: "param-slider", description: "参数调节滑块", type: "normal" }
        ],
        GUIDEBOOK: `这是[页面名称]页面。
        页面布局：
        - 顶部为参数设置区
        - 中间为可视化图像
        使用流程：
        1. 首先在下拉框选择算法...
        2. 调节滑块设置初始值...
        3. 点击开始按钮查看结果...`
    };
</script>
<!-- 如果有页面专用的 JS，也在此处引入 -->
<script src="/static/js/your-page-logic.js"></script>
{% endblock %}
```

### 总结
完成上述配置后，网页加载时 `ai-sidebar.js` 会自动读取 `window.AI_CONFIG`。当用户与侧栏交互时，脚本会将这些信息发送至后端。后端 Agent 会优先查找文档库，若无匹配文档则根据 `GUIDEBOOK` 和 `BUTTON_REGISTRY` 进行推理，最终指导前端完成文字回复和控件高亮。
