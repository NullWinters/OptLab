在实验笔记模块解耦后，为一个新网页添加实验笔记功能非常简单，主要分为两个步骤：**继承母版** 和 **配置实验笔记参数**。

以下是具体操作流程及代码示例：

### 1. 继承基础母版 (base.html)
新网页必须使用 Jinja2 的模板继承机制。通过继承 `base.html`，网页会自动引入实验笔记模块所需的资源。

默认情况下，实验笔记挂载点位于页面底部。如果你需要**自定义实验笔记在页面中的位置**（例如放入特定的容器中以对齐宽度），可以覆盖 `experiment_notes` block：

```html
{% extends 'base.html' %}

{% block title %}实验页面标题{% endblock %}

{# 1. 覆盖默认 block 为空，移除原位置的挂载点 #}
{% block experiment_notes %}{% endblock %}

{% block content %}
<div class="container">
    <!-- 页面上方内容 -->
    <div class="main-layout">...</div>

    {# 2. 在你需要的位置插入挂载点 #}
    {% include 'components/_experiment_notes.html' %}
    
    <footer>页脚信息</footer>
</div>
{% endblock %}
```

### 2. 配置 window.EXPERIMENT_NOTES_CONFIG
在 `{% block scripts %}` 中定义 `window.EXPERIMENT_NOTES_CONFIG` 对象。该配置告诉模块如何标识当前实验，以及如何获取当前页面的实验数据。

该对象包含以下关键字段：
*   **`experimentKey`**: 实验的唯一标识符（字符串）。用于在后端存储和检索该实验的笔记。
*   **`getPageData`**: 一个可选的函数。当用户点击“AI 生成笔记”时，该函数会被调用，其返回的对象将作为上下文发送给 AI。你可以在此处收集当前页面的参数、状态或实验结果。

```html
{% block scripts %}
<script>
    window.EXPERIMENT_NOTES_CONFIG = {
        // 1. 设置实验唯一 ID
        experimentKey: 'your_experiment_unique_id',
        
        // 2. (可选) 定义如何获取当前页面数据，供 AI 生成笔记参考
        getPageData: function () {
            // 假设页面上有一个全局状态对象 AppState
            var st = window.AppState;
            if (!st) return {};
            return {
                algorithm: st.currentAlgorithm,
                parameters: st.params,
                results: st.results_log
            };
        }
    };
</script>
<!-- 页面专用的 JS 逻辑 -->
<script src="/static/js/your-page-logic.js"></script>
{% endblock %}
```

### 总结
完成上述配置后，网页加载时 `experiment-notes.js` 会自动读取 `window.EXPERIMENT_NOTES_CONFIG` 并完成 UI 挂载。
*   **自动保存**：用户输入的笔记会根据 `experimentKey` 自动关联并保存。
*   **行为追踪**：模块会自动尝试监听页面上的常见交互（如算法切换、参数滑动等），并记录在行为序列中，供 AI 生成笔记时参考。
*   **AI 生成**：若配置了 `getPageData`，AI 在生成笔记时将结合页面实时状态，提供更具针对性的总结。
